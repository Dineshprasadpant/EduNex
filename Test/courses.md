# Module Test Cases: Courses & Categories Management

This document contains detailed test cases for the **Courses & Categories Management** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **CRS-001** | Create Course with HTML formatted Description containing script tags (XSS check) | `CreateCourseRequestDto` with `Description = "<h3>Intro</h3><script>alert('xss')</script>"` | Return HTTP 201 Created but the `<script>` tag must be stripped while keeping safe elements like `<h3>`. | Verified in `CourseService.cs` line 44: Uses Ganss `HtmlSanitizer` with configured whitelist. Sanitizer strips the script block completely, saving only safe HTML to the database. | **PASS** |
| **CRS-002** | Create Course and verify automatic unique Slugification | `CreateCourseRequestDto` with `Title = "IOE Prep Course!"` | Return HTTP 201 with generated `Slug = "ioe-prep-course"`. | Verified in `CourseService.cs` line 111: `Slugify` uses regex `[^a-z0-9]+` to generate standard slug. | **PASS** |
| **CRS-003** | Create Course with identical title to an existing course (Unique Slug collision check) | `CreateCourseRequestDto` with `Title = "IOE Prep Course!"` when `ioe-prep-course` slug already exists | Generated slug should automatically append a counter suffix (e.g. `"ioe-prep-course-1"`). | Verified in `CourseService.cs` line 124: `GenerateUniqueSlugAsync` queries existing slugs, detects collision, and enters loop appending `-1`, `-2`, etc. until uniqueness is resolved. | **PASS** |
| **CRS-004** | List public-facing Courses | GET request to `/api/courses` with pagination and filters | Returns list of Active courses only, matching filters, and excludes sensitive `Information` columns. | Verified in `CourseService.cs` line 133 and `CourseDal.cs`: sets `ActiveOnly = true` inside `CourseFilters` and fetches public `CourseListDto`. | **PASS** |
| **CRS-005** | List Admin/Staff Courses (Full List) | GET request to `/api/courses/admin/all` (Authorized as Admin) | Returns list of all courses (both Active and Inactive) for management. | Verified in `CourseService.cs` line 151: calls `ListAllCoursesAsync` which sets `ActiveOnly = false` to include hidden draft courses. | **PASS** |
| **CRS-006** | Get Single Course by ID (Admin scope) | GET request to `/api/courses/{id}` as Admin | Returns full details of the course including the sensitive `Information` field meant for course-enrolled/admin eyes only. | Verified in `CourseService.cs` line 169: calls `GetCourseByIdAsync` which executes Dal query returning the full `CourseDetailDto` containing the `Information` field. | **PASS** |
