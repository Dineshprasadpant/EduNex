# Module Test Cases: Communications, Feedback & Interactive Features

This document contains detailed test cases for the **Communications, Feedback & Interactive Features** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **COM-001** | Submit Public Contact Form | POST `/api/contact` with Name, Email, Subject, and Message. Guarded by Cloudflare Turnstile. | Return HTTP 201 Created. Message saved, email notification immediately sent to Admin mailbox. | Verified in `ContactService.cs` and `ContactController.cs`: saves message, and invokes `_mail.SendNewContactNotificationAsync` to alert the admin. | **PASS** |
| **COM-002** | Submit Contact Form with invalid inputs | POST `/api/contact` with message shorter than 20 characters, or invalid email format | Returns HTTP 400 Bad Request or 422 Validation Error. | **COM-003** | Reply to Contact message (Admin scope) | POST `/api/contact/{id}/reply` with reply text | Return HTTP 200 OK.
