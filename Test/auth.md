# Module Test Cases: Authentication & Authorization (Auth)

This document contains detailed test cases for the **Authentication & Authorization** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **AUT-001** | Student Registration with missing Citizenship certificate | `RegisterRequestDto` with `Citizenship = null` | Return HTTP 400 Bad Request with "Citizenship certificate is required" message. User row should not be created. | C# `AuthService.RegisterAsync` checks `request.Citizenship is null` and throws `BadRequestException("Citizenship certificate is required")` before any DB interaction or hashing. | **PASS** |
| **AUT-002** | Student Registration on Paid Plan with missing Payment Receipt | `RegisterRequestDto` with `Plan = "paid"`, `Payment = null` | Return HTTP 400 Bad Request with "Payment image is required for paid plans" message. No DB rows created. | C# `AuthService.RegisterAsync` checks `request.Plan != PlanType.Free && request.Payment is null` and throws `BadRequestException`. Verified that file upload is short-circuited. | **PASS** |
| **AUT-003** | Student Registration with existing email address | `RegisterRequestDto` with email `existing@example.com` | Return HTTP 409 Conflict with "A user with this email already exists" message. | C# `AuthService.RegisterAsync` executes `_authDal.FindUserByEmailAsync(request.Email)` at line 44, detects duplicate, and throws `ConflictException`. | **PASS** |
| **AUT-004** | Student Registration with existing phone number | `RegisterRequestDto` with phone `+977-9812345678` | Return HTTP 409 Conflict with "A user with this phone number already exists" message. | C# `AuthService.RegisterAsync` executes `_authDal.FindUserByPhoneAsync` at line 47, detects duplicate, and throws `ConflictException`. | **PASS** |
| **AUT-005** | Successful Student Registration (Happy Path) | `RegisterRequestDto` with valid details, Citizenship PDF, Payment Image (for Paid Plan) | HTTP 201 Created with successful registration message. Document files uploaded to storage, password hashed with BCrypt, User + StudentProfile tables updated. Admin email notification sent. | Verified in `AuthService.RegisterAsync`: files uploaded via `_fileService.UploadFileAsync`, password hashed using BCrypt, transactional creation of User and StudentProfile, and `SendNewUserAdminNotificationAsync` triggered. | **PASS** |
| **AUT-006** | Successful Login with correct credentials | `LoginRequestDto` with valid `Email` and `Password` | HTTP 200 OK with `AccessToken`, `RefreshToken`, and `UserDto` (without `PasswordHash` leaked). `FailedLoginAttempts` reset to 0 in DB. | Verified in `AuthService.LoginAsync` and `AuthController.Login`. Executes `_authDal.RecordSuccessfulLoginAsync`, signs AccessToken with expiration, hashes and persists RefreshToken, returns token pair and user profile. | **PASS** |
| **AUT-007** | Login attempt with incorrect password (Failed attempt tracking) | `LoginRequestDto` with correct Email but wrong Password | HTTP 401 Unauthorized with "Invalid email or password" message. Failed login count incremented in database. | Verified in `AuthService.LoginAsync` line 133: BCrypt verification fails, increments `FailedLoginAttempts` and persists via `RecordFailedLoginAsync`. | **PASS** |
| **AUT-008** | Login attempt leading to account lockout | 5 consecutive failed login attempts on an account | HTTP 403 Forbidden with "Account is locked" message. Account marked as locked in DB. | Verified in `AuthService.LoginAsync` line 134: `shouldLock` evaluates to true when attempts exceed `_maxFailedLoginAttempts` (default 5). Calls `_authDal.RecordFailedLoginAsync` with `locked = true`. Subsequent logins are blocked at line 125. | **PASS** |
| **AUT-009** | Login with unverified account | `LoginRequestDto` with correct credentials but account `IsVerified = false` | HTTP 403 Forbidden with "Account not verified" message. No token issued. | Verified in `AuthService.LoginAsync` line 147: check for `!user.IsVerified` is performed after password validation, returning `Fail` with `isAccountIssue = true` which translates to 403 in Controller. | **PASS** |
| **AUT-010** | Login with blocked account | `LoginRequestDto` with correct credentials but account `IsBlocked = true` | HTTP 403 Forbidden with "Account is blocked" message. No token issued. | Verified in `AuthService.LoginAsync` line 128: blocks at the entry gate of Login validation before password checking is performed. | **PASS** |
| **AUT-011** | Request Interception for Blocked User | Token-authenticated request made by a user who has just been blocked | HTTP 401 Unauthorized with "Your account has been blocked" message. | Verified in `BlockedUserCheckFilter.cs` line 29: checks user state from `IAuthDal.GetUserAuthStateAsync(userId)`. If `state.IsBlocked` is true, throws `UnauthorizedException` which short-circuits request. | **PASS** |
| **AUT-012** | Cloudflare Turnstile Token Verification (Missing Token) | Post Request to `/register` without captcha token | HTTP 400 Bad HttpRequest with "Captcha verification required." message. | Verified in `VerifyTurnstileAttribute.cs` line 29: checks form or header for `turnstileToken` / `cf-turnstile-response`. If missing, throws `BadHttpRequestException` immediately. | **PASS** |
| **AUT-013** | Cloudflare Turnstile Token Verification (Invalid Token) | Post Request to `/register` with an expired/invalid Turnstile token | HTTP 400 Bad HttpRequest with "Captcha verification failed. Please try again." message. | Verified in `VerifyTurnstileAttribute.cs` line 44: makes POST request to Cloudflare Challenges endpoint. When `result.Success` is false, throws `BadRequestException`. | **PASS** |
| **AUT-014** | Silent Turnstile Captcha Bypass in Dev/Test | Post Request to `/register` with Turnstile secret omitted in configuration | Skip Captcha verification and proceed with registration. | Verified in `VerifyTurnstileAttribute.cs` line 20: if configuration key `Turnstile:SecretKey` is empty or null, calls `next()` and executes action. | **PASS** |
| **AUT-015** | Refresh Token Rotation (Happy Path) | POST request to `/refresh` with active, unexpired RefreshToken | HTTP 200 OK with newly minted AccessToken and rotated RefreshToken. Old token marked as revoked. | Verified in `TokenService.RotateRefreshTokenAsync`: old token row updated to `IsRevoked = true` and a new token pair generated and returned. | **PASS** |
| **AUT-016** | Refresh Token Rotation with Revoked Token | POST request to `/refresh` with a previously used/revoked RefreshToken | HTTP 401 Unauthorized with "Invalid or expired refresh token" message. | Verified in `TokenService.cs`: database lookup for unrevoked tokens returns null; throws `UnauthorizedException`. | **PASS** |
| **AUT-017** | Token-Authenticated Get Current User profile (GetMe) | GET request to `/auth/me` with valid Authorization header | HTTP 200 OK with `UserDto` corresponding to the authenticated token. | Verified in `AuthController.GetMe` and `AuthService.GetMeAsync`: parses `ClaimTypes.NameIdentifier` from current claims, fetches user from DB, and returns DTO. | **PASS** |
