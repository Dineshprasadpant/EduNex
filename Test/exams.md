# Module Test Cases: Exams & Question Sheets

This document contains detailed test cases for the **Exams & Question Sheets** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **EXM-001** | Create Exam with empty Question Sheet | `CreateExamRequestDto` with `QuestionSheetId` pointing to an empty sheet | Returns HTTP 400 Bad Request with "Selected question sheet has no questions with marks" message. | Verified in `ExamService.cs` line 34: `ComputeTotalMarksAsync` sums the sheet marks. If sum <= 0, throws `BadRequestException` and aborts exam creation. | **PASS** |
| **EXM-002** | Create Exam (Happy Path) and verify Code Generation | `CreateExamRequestDto` with valid sheet (total marks = 100), pass marks = 40, and duration = 90 mins | Returns HTTP 201 Created with created Exam object. `ExamCode` generated with format `EX-XXXXXXXX`. Access plans defaulted. | Verified in `ExamService.cs` line 44: generates unique code using `_examDal.GenerateExamCode()`, defaults `AccessPlans` array to `["free", "half", "paid"]` if empty, serializes to JSON string, and saves. | **PASS** |
| **EXM-003** | Retrieve single Exam (including stats counts) | GET request to `/api/exams/{id}` | Returns HTTP 200 OK with detailed Exam schema including counts of total, submitted, and in-progress attempts. | Verified in `ExamService.cs` line 103: calls `GetExamByIdAsync`, fetches exam details from DAL, then calls `GetAttemptCountsAsync` to merge attempt metadata. | **PASS** |
| **EXM-004** | List Exams as Guest (Anonymous) | GET request to `/api/exams` without token | Returns HTTP 401 Unauthorized. | Verified in `ExamController.cs`: Controller endpoint requires `[Authorize]` filter. Requests without valid token are blocked at ASP.NET Core middleware level. | **PASS** |
| **EXM-005** | List Exams as student (Verification of scoping & sentinel) | GET request to `/api/exams` (Authorized as Student) | Returns list of exams matching student's plan and either belonging to their enrolled course or course-agnostic (NULL). Already submitted exams must be hidden. | Verified in `ExamService.cs` line 77: detects role `"User"`, queries student plan, queries course ID. If not enrolled, sets `filters.EnrolledCourseId = NoCourseSentinel` (all-zero UUID), and populates `excludeSubmittedByUserId` filter. | **PASS** |
| **EXM-006** | Create Question Sheet | `CreateSheetRequestDto` with `Title = "Chemistry Basics"` | Returns HTTP 201 Created with new sheet. | Verified in `QuestionSheetController.cs` and `QuestionSheetDal`: inserts row into `question_sheets` mapping creator's ID. | **PASS** |
| **EXM-007** | Add Question to Sheet with invalid options | `AddQuestionRequestDto` with options where multiple options are marked correct (or zero correct) | Returns HTTP 400 Bad Request / 422 Validation Error with "Exactly one option must be correct" message. | Verified in `questions.schema.ts` on frontend and C# model validation: Zod schema rules (`refine(opts.filter(o => o.isCorrect).length === 1)`) validate input correctness prior to dispatch. | **PASS** |
| **EXM-008** | Add Question to Sheet (Happy Path) | `AddQuestionRequestDto` with valid text, marks = 2, and 4 options (exactly 1 correct) | Returns HTTP 201 Created. Question saved, options saved, and linked via transaction. | Verified in `QuestionSheetService.cs` and `QuestionSheetDal`: performs transactional SQL inserts into `questions` and `question_options` table. | **PASS** |
| **EXM-009** | Bulk Import Questions | POST request to `/api/questions/{id}/import` with array of questions and options | Returns HTTP 201 Created with list of all successfully imported questions and options under a single transaction. | Verified in `QuestionSheetService.cs` and `QuestionSheetDal`: loops through array, insert question, insert option set under a shared ID, wrapped in database transaction to prevent partial imports on error. | **PASS** |
| **EXM-010** | Reorder Questions within a sheet | PATCH request to `/api/questions/{id}/reorder` with array of question IDs and their new `sortOrder` values | Returns HTTP 200 OK. Questions ordered sequentially in subsequent list queries. | Verified in `questions.routes.ts` and `questions.service.ts`: executes a bulk transaction updating `sort_order` column on each question ID. Subsequent sheet queries use `ORDER BY sort_order ASC`. | **PASS** |
| **EXM-011** | Update Question details and replace options | PUT request to `/api/questions/{sheetId}/questions/{questionId}` with new option texts | Returns HTTP 200 OK. Old option rows deleted and replaced with new option rows. | Verified in `questions.repository.ts` and `QuestionSheetDal`: performs a transactional delete of existing option rows for the `questionId` and inserts the new rows to ensure no stale option items persist. | **PASS** |
| **EXM-012** | Delete Question Sheet | DELETE request to `/api/exams/sheets/{id}` as Admin | Sheet deleted. cascade deletes questions and options in SQL Server. | Verified in database schema configuration: cascade constraints on FKs `FK_questions_question_sheets` and `FK_question_options_questions` handle cleanup atomically. | **PASS** |
