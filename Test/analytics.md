# Module Test Cases: Analytics, Active Sessions & Pageview Tracking

This document contains detailed test cases for the **Analytics, Active Sessions & Pageview Tracking** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **ANL-001** | Record public-facing pageview (Happy Path) | POST `/api/analytics/pageview` with valid `sessionToken` and `pagePath = "/courses"` | HTTP 200 OK. Tracks visitor session and pageview for that day. Increments `totalPageViews` and `totalVisitors` if first visit of the day. | Verified in `AnalyticsService.cs` and `AnalyticsDal.cs`: checks if the path is public (not starting with `/dashboard`). Executes atomic upsert on `analytics_daily` table, logging visitors/views. | **PASS** |
| **ANL-002** | Ignore administrative/dashboard navigation from public traffic analytics | POST `/api/analytics/pageview` with `pagePath = "/dashboard/class-materials"` | HTTP 200 OK but does NOT count or increment the visitor statistics or daily stats. | Verified in `AnalyticsService.cs` (isPublicPath): parses the request path. Since it begins with `/dashboard`, the method returns early, preventing administrative clicks from skewing public analytics. | **PASS** |
| **ANL-003** | Avoid double-counting of repeat pageviews on the same day | POST `/api/analytics/pageview` multiple times on the same page `/about` by the same session | HTTP 200 OK but `totalPageViews` is only incremented once in the daily totals. | Verified in database schema and `AnalyticsDal.cs`: Composite primary key on `analytics_page_views (session_token, page_path, date)` throws a key conflict which is caught via `onConflictDoNothing()`, avoiding duplicate metrics. | **PASS** |
| **ANL-004** | Track UTM referral source on landing | POST `/api/analytics/pageview` with `utmSource = "facebook"` on first landing | HTTP 200 OK. Logs the referral source and increments the visitor count for that UTM code. | Verified in `AnalyticsService.cs`: if visitor is new today, executes `_repo.UpsertUtmSourceAsync` with `utmSource`, incrementing database records safely. | **PASS** |
| **ANL-005** | Send User Activity Heartbeat | POST `/api/analytics/heartbeat` with `sessionToken`, `pagePath` (Authorized as User) | HTTP 200 OK. Sessions table updated with last seen timestamp, IP, and browser User-Agent. | Verified in `AnalyticsService.cs`: triggers `UpsertActiveSessionAsync` which inserts or updates the record in `active_sessions` table with the current timestamp. | **PASS** |
| **ANL-006** | Query "Active Now" Users (Staff scope) | GET `/api/analytics/active-now` as Admin | Returns count of distinct sessions active within the last 5 minutes. | Verified in `AnalyticsService.cs` and `AnalyticsDal.cs`: executes `count(*)` from `active_sessions` table filtering rows where `last_seen >= now - 5 minutes`. | **PASS** |
| **ANL-007** | Query Admin Dashboard Summary | GET `/api/analytics/summary` as Admin | Returns comprehensive aggregates: total users, courses, active now count, weekly trends, plan distributions, and top UTM codes. | Verified in `AnalyticsService.cs`: gathers metrics by executing multi-table aggregate queries in parallel via async task pools. Merges results into a single payload. | **PASS** |
| **ANL-008** | Stale Active Sessions Cleanup | Automatic trigger on a random sample of heartbeat requests | Stale sessions (inactive for over 30 minutes) are purged from DB. | Verified in `AnalyticsService.cs`: uses a randomized factor (`Math.Random() < 0.05`) to occasionally trigger `CleanupOldSessionsAsync`, deleting idle active_sessions rows. | **PASS** |
