# Module Test Cases: Exam Attempts, Scoring & Performance

This document contains detailed test cases for the **Exam Attempts, Scoring & Performance** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **ATT-001** | Start Exam Attempt (First time, happy path) | POST `/api/exam-attempts/exams/{examId}/start` inside valid window | Return HTTP 201 Created. Creates a new `InProgress` attempt. Returns questions and options *without* answers/correct marks leaked. | Verified in `ExamService.cs` line 241: `StartAttemptAsync` checks window, calls `CreateAttemptAsync`, strips `IsCorrect` values from the loaded `QuestionOption` collection, and returns safe data. | **PASS** |
| **ATT-002** | Resume an Active/In-Progress Attempt | POST `/api/exam-attempts/exams/{examId}/start` for a student with an active draft | Returns same attempt ID with previously saved answers and flagged questions to resume. | Verified in `ExamService.cs` line 210: finds active attempt via `FindOpenAttemptAsync`, loads answers, and maps them to `existingAnswers` dictionary. | **PASS** |
| **ATT-003** | Prevent multiple submissions of the same exam | POST `/api/exam-attempts/exams/{examId}/start` when a student has already submitted | Return HTTP 200 OK with `AlreadySubmitted = true` and details of prior attempt to prevent re-taking. | Verified in `ExamService.cs` line 180: checks `FindSubmittedAttemptAsync` first. If present, returns `AlreadySubmittedResult` without issuing fresh attempt. | **PASS** |
| **ATT-004** | Course ID Gate check on start attempt | POST `/api/exam-attempts/exams/{examId}/start` for an exam restricted to Course A, by a student enrolled in Course B | Return HTTP 403 Forbidden with "This exam is restricted to students of a different course" message. | Verified in `ExamService.cs` line 191: if `exam.CourseId` is configured, calls `FindStudentCourseIdAsync` and compares, short-circuiting if not matching. | **PASS** |
| **ATT-005** | Save Answer to Question during attempt (Normal action) | POST `/api/exam-attempts/{attemptId}/answer` with `questionId` and `selectedOptionId` | Returns HTTP 200 OK. Save answer to DB, resolve correctness server-side but do NOT leak `isCorrect` to the student. | Verified in `ExamService.cs` line 265: calls `AssertActiveAttemptAsync` and `AssertExamInWindowAsync`. Resolves correctness using `FindOptionCorrectnessAsync` and saves. Strips correctness field in return. | **PASS** |
| **ATT-006** | Save Answer after exam window has closed (Auto-Submit test) | POST `/api/exam-attempts/{attemptId}/answer` after `exam.EndDateTime` has passed | Return HTTP 400 Bad Request with "Exam time has expired -- your attempt has been auto-submitted" message. Attempt status updated to submitted. | Verified in `ExamService.cs` line 253: checks window in `AssertExamInWindowAsync`. If `EndDateTime` is in the past, it automatically triggers `SubmitAttemptAsync` and throws `BadRequestException`. | **PASS** |
| **ATT-007** | Flag/Unflag Question for review | POST `/api/exam-attempts/{attemptId}/flag` with `questionId` and `isFlagged = true` | Returns HTTP 200 OK with updated flagged state. | Verified in `ExamService.cs` line 282: invokes `_examDal.FlagQuestionAsync` to toggle flagging, returning current status. | **PASS** |
| **ATT-008** | Submit Exam Attempt (Scoring & Negative marking verification) | POST `/api/exam-attempts/{attemptId}/submit` for attempt with 15 correct (+15 marks), 5 incorrect answers, on exam with 20% negative marking | Return HTTP 200 OK with score summary. Total positive score = 15. Negative deduction = 20% of 5 = 1 mark. Final score = 14. Percentage and pass/fail computed. | Verified in `ExamService.cs` line 361: triggers `SubmitAttemptAsync`. Calculates `tallied.PositiveMarks` and `tallied.IncorrectMarks`. Computes `negativeDeducted` as `(negPercent / 100m) * tallied.IncorrectMarks`. Saves record. | **PASS** |
| **ATT-009** | Double Submit Protection | POST `/api/exam-attempts/{attemptId}/submit` for already submitted attempt | Return HTTP 409 Conflict with "This attempt has already been submitted" message. | Verified in `ExamService.cs` line 357: checks `attempt.Status == ExamAttemptStatus.Submitted` and throws `ConflictException` if true. | **PASS** |
| **ATT-010** | Retrieve Student Attempt Detail (Post-Submit view) | GET `/api/exam-attempts/{attemptId}` as the Student | Returns HTTP 200 OK with full question sheet, options, selected answers, and the correct answer key now visible. | Verified in `ExamService.cs` line 448: triggers `GetAttemptDetailAsync`. | **ATT-011** | Retrieve Student Attempt Detail (Pre-Submit sneak-peek prevention) | GET `/api/exam-attempts/{attemptId}` as Student while attempt is still In-Progress | Returns HTTP 200 OK but correct answers and correctness fields must be stripped/nullified. | Verified in `ExamService.cs` line 454: `allowAnswerKey` is false because attempt status is not submitted. | **ATT-012** | Leaderboard Ranking computation | GET `/api/leaderboard` with `examId` | Returns ranked student rows based on percentage, score descending, and time taken ascending. | Verified in `leaderboard.repository.ts` line 20: uses raw SQL CTE window function `ROW_NUMBER() OVER (ORDER BY ea.percentage DESC, ea.marks_obtained DESC, ea.time_taken_seconds ASC)` to compute ranks. | **PASS** |
| **ATT-013** | Leaderboard Status Pass/Fail verification | GET `/api/leaderboard` with `examId` | Returns student rows, and includes dynamic pass/fail labels depending on whether score >= pass_marks. | Verified in `leaderboard.repository.ts` line 34: uses `CASE WHEN score::numeric >= pass_marks::numeric THEN 'pass' ELSE 'fail' END`. | **PASS** |
