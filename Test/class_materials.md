# Module Test Cases: Class Materials, File Uploads & S3 Integration

This document contains detailed test cases for the **Class Materials, File Uploads & S3 Integration** module of the EduNex platform, spanning the Next.js `Dragon-frontend` and the C# `.NET` backend (`EduNex.API` & `EduNex.Services`). The results are verified via a detailed "disk checking" of the implementation logic.

## Tabular Test Cases & Results

| Test Case ID | Action | Input | Expected Result | Actual Result (Disk Checked) | Test Result |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **MAT-001** | Create Class Material (Happy path, Staff scope) | `CreateClassMaterialRequest` with a valid `MediaId` and `CourseId` | Returns HTTP 201 Created. Class material created, email notification dispatched to all students enrolled in that course. | Verified in `ClassMaterialService.cs`: updates the database, and calls `_mail.SendClassMaterialAddedEmailAsync` in a background thread with the materials page link. | **PASS** |
| **MAT-002** | Fetch Materials List as Un-enrolled Student | GET `/api/class-materials` as student with no active course enrollment | Return HTTP 200 OK with empty array `[]` (not throwing error). | Verified in `ClassMaterialService.cs` line 67: `ResolveCourseFilterAsync` returns `emptyForStudent = true` if student has no course ID, short-circuiting to return empty list. | **PASS** |
| **MAT-003** | Fetch Materials List as Enrolled Student (Scope check) | GET `/api/class-materials` (Authorized as Student of Course A) | Returns materials belonging to Course A only. Materials from Course B are automatically filtered out. | Verified in `ClassMaterialService.cs` line 80: `ResolveCourseFilterAsync` fetches `studentCourseId` via `FindStudentCourseIdAsync(userId)`, overrides search filter, returning scoped rows. | **PASS** |
| **MAT-004** | Prevent Unauthorized Access to other Course's Materials | GET `/api/class-materials/{id}` for Course B material, by student of Course A | Return HTTP 403 Forbidden with "You do not have access to this material" message. | Verified in `ClassMaterialService.cs` line 99: `AssertCanAccessAsync` checks `studentCourseId`. If mismatch, throws `ForbiddenException`. | **PASS** |
| **MAT-005** | Sanitize Out Storage URLs from public metadata API | GET `/api/class-materials/{id}` as a student | Returns class material details, but completely strips the raw S3 bucket URL / file storage path from the output. | Verified in `ClassMaterialService.cs` line 61: calls `Sanitize` helper which clones the DTO and completely removes the `FileUrl` and `S3Key` properties before returning, making sure students can't direct-link without permission. | **PASS** |
| **MAT-006** | Request File Download Presigned URL (Authorized) | GET `/api/class-materials/{id}/download` as Admin | Returns a short-lived presigned URL (300-second TTL) with `Content-Disposition: attachment; filename="{originalName}"`. | Verified in `ClassMaterialService.cs` (DownloadAsync): loads the associated `media` row, calls `_fileStorage.GetPresignedDownloadUrl` specifying `DownloadUrlTtlSeconds = 300` and the original filename. | **PASS** |
| **MAT-007** | Request File View Inline Presigned URL (Authorized) | GET `/api/class-materials/{id}/view-url` as Student of that course | Returns an inline view-friendly signed URL (1-hour TTL) with correct headers for direct in-app display. | Verified in `ClassMaterialService.cs` (ViewUrlAsync): checks access, generates a presigned view URL with a 3600-second TTL and `Content-Disposition: inline` through the S3 service. | **PASS** |
| **MAT-008** | Stream PDF via Same-Origin Proxy (CORS bypass for viewer) | GET `/api/class-materials/{id}/stream` | Streams bytes directly from S3 bucket through the .NET backend API with `Content-Disposition: inline`, allowing pdf.js on localhost:3000 to render without CORS rules. | Verified in `ClassMaterialService.cs` (StreamAsync) and `ClassMaterialController.cs`: gets object stream using `_fileStorage.GetObjectStreamAsync` and pipes it directly into the `HttpContext.Response.Body`. | **PASS** |
| **MAT-010** | Multipart Upload via S3 | POST `/api/files/upload` with form file | Uploads buffer to configured S3 bucket with clean sanitised filename and returns Media ID to relate it to Course/Announcement. | Verified in `FileService.cs` and `FilesController.cs`: accepts up to 50MB files, uses `AmazonS3Client` to write object, saves metadata in `media` database, and outputs details. | **PASS** |
